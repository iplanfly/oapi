必读

域名

域名暂不可用，以IP 123.206.84.232 替代之。

请求方式

每个API都需以指定请求方式请求，否则会得到 405 (Method Not Allowed) 的拒绝。

权限验证

对于绝大多数的请求，都需要在 URL 中带入 名为 access-token 的参数，其值为 登录和 注册 成功所返回的字符串，这类似于浏览器的登录。

大道至简

通常情况下，服务端仅仅返回客户端最为需要的内容，比如 登录 之后的 access-token 。而是否成功的判断可以参考 返回体 中的状态码。这意味着，当状态码不是 200 的时候它一定发生了错误。错误信息相比则详细些，他是一个 json 格式的字符串。如下是一次注册失败的例子：
{
    "name": "Forbidden",
    "message": "手机号不能为空。",
    "code": 0,
    "status": 403
}
其中 name 是对本次请求状态码的短描述，其使用着 HTTP 协议的推荐用语，message 则说明了为什么会失败。code 含义暂未使用且待定，status 则是状态码（和返回header中状态码相同）。

用户User

注册signup

地址：/user/signup

请求方式：post

参数：
# 参数名 类型 描述 必选 约束
1 mobilePhone integer 手机号 是 11位
2 password string 密码 是 最低6位
3 passwordRepeat string 重复密码 是 必须和密码相同
4 invitationCode string 邀请码 是 8位
返回值：
结果 类型 描述
成功 string 登陆凭证
失败 string/json 错误信息

编辑个人信息edit-info

地址：/user/edit-info?access-token=xxx

请求方式：post

参数：
# 参数名 类型 描述 必选 约束

2 face file 头像 否 png 或者 jpg格式
3 qq integer QQ号 否 不超过15位
4 wechat string 微信号 否 不超过60位
5 qqGroup integer QQ群 否 不超过15位
6 intro string 个人介绍 否 不超过60位
返回值：
结果 类型 描述
成功 string true
失败 string/json 错误信息
